Runtime Introduction

Knowledgebases that you create in the KnowledgeWright Workshop can run in a variety of environments and applications. KnowledgeWright includes a ready-to-run web server interface for the Basic Jig, and prototype interfaces for Java, Delphi and Visual Basic.

All these interfaces are built using the KnowledgeWright Interface, KWI. This is an API for running knowledgebases. The KWI provides calls to start and end reasoning sessions, assert new facts and retrieve solutions. The details of the KWI are covered in the Overview and Reference.

Redistributable Runtime Files

If you are running your knowledgebases on a web server, you may install the following files:

kwcgi{jig}(.exe), kwcgi{jig}.xpl, and kwcgirun.plm (which is the compiled form of kwcgirun.pro that you may modify), amzi.dll (Windows), libamzi.so (Unix), aosutils.lsx, aodbc.lsx, amzi.cfg.

If you are running your knowledgebases as part of an application, you may redistribute the following files with your application:

{jig}.xpl, amzi.dll (Windows), libamzi.so (Unix), aosutils.lsx, aodbc.lsx, amzi.cfg, amzijni.dll (Windows, Java) or libamzijni.so (Unix, Java).

{jig} is one of basic, support, sales or a custom jig provided to you by Amzi!
